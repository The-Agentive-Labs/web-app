Pattern

Guided Templates

Generate Structure

Help users produce reliable outputs without needing to understand the system internals.

Learn

What This Is

Guided Templates are structured frameworks that constrain user input to produce consistent, predictable outputs. They act as scaffolding—providing clear fields, constraints, and context that shape the creation process without requiring users to understand underlying logic or architecture.

Problem Framing

Users often struggle to translate intent into structured outputs, especially when working with unfamiliar tools or complex workflows. Without guidance, they may produce incomplete, inconsistent, or incorrect results. Guided templates reduce cognitive load by providing clear input boundaries and expected output structure.

When to Use / Avoid

Use When

  • Outcomes must be consistent
  • Users are new to the system
  • Mistakes are costly or time-consuming
  • The goal is clearly defined
  • There's a known "right answer" structure

Avoid When

  • Exploration matters more than correctness
  • Intent is undefined or evolving
  • Rigid structure would block creativity
  • Users need full control
  • Output requirements vary significantly
Remember: Templates are scaffolding, not shortcuts. They guide creation but should not replace understanding or judgment.

Build

Template Structure

A guided template consists of clear input fields, validation rules, and expected output format.

Field Description
Goal Statement What the user is trying to accomplish
Context Inputs Required information to guide generation
Constraints Boundaries or rules that must be followed
Output Format Expected structure of the final result
Validation Rules Checks to ensure output quality

Example Output

{ "goal": "Create a user research plan", "context": { "audience": "B2B product managers", "timeline": "2 weeks", "resources": "Remote interviews" }, "output": { "research_questions": [...], "methodology": "...", "participant_criteria": [...], "timeline": [...] } }

Design Considerations

  • Progressive disclosure: Show fields incrementally to avoid overwhelming users
  • Clear labels: Use plain language to describe what each field requires
  • Inline help: Provide contextual guidance without leaving the template
  • Validation feedback: Show errors and corrections in real-time
  • Preview mode: Allow users to see output before finalizing

Risks & Tradeoffs

Risk: Over-constraining templates can stifle creativity and prevent users from exploring alternative approaches.

Mitigation: Provide an "advanced" or "custom" mode that allows users to bypass template constraints when needed.
Tradeoff: Templates optimize for consistency at the cost of flexibility. Users may feel restricted if templates don't accommodate their specific use case.

Consideration: Offer multiple template variants or allow field customization to balance structure with adaptability.
